# Thinkmate — Your Thinking Partner

Chrome extension that acts as a universal AI coaching layer in the browser. Coaches the thinking behind your words, not just grammar. Works on any website.

## Core Architecture

### Three Layers

1. **Content Script** (`content.js` + `/core`) — injected into every page. Detects input fields, renders trigger button and panel via Shadow DOM, captures text.
2. **Background Service Worker** (`background.js`) — handles all API calls (avoids CORS), manages provider routing, model discovery.
3. **Options Page** (`options.html/js`) — provider config, model selection, coach management, UI preferences.

### Coach System — Instruction-Based Plugins

All coaches are configuration-based, not code-based. Each coach is a JSON/JS config file exporting:

```js
{
  id: "english-coach",
  name: "English Coach",
  description: "Grammar, vocabulary, pronunciation with Color Vowel, IELTS-level feedback",
  icon: "🇬🇧",
  enabled: true,
  systemPrompt: "...",
  outputSchema: {
    // Defines the JSON structure the AI must return
    tabs: [
      { key: "corrected", label: "Corrected", type: "text" },
      { key: "grammar", label: "Grammar", type: "list-of-cards", fields: ["original", "fix", "explanation"] },
      { key: "vocabulary", label: "Vocabulary", type: "list-of-cards", fields: ["original", "suggestion", "reason"] },
      { key: "pronunciation", label: "Pronunciation", type: "list-of-cards", fields: ["word", "phonetic", "stress", "color_vowel", "tip"] },
      { key: "practice_prompt", label: "Practice", type: "text" }
    ]
  },
  settings: {
    ielts_target: { type: "select", options: ["6.0", "7.0", "8.0"], default: "8.0" },
    english_variant: { type: "select", options: ["American", "British"], default: "American" }
  }
}
```

**Adding a new coach** = one new config file in `/coaches` + one import line in `/coaches/index.js`.

**User-created custom coaches** use the same system — user provides name, icon, instruction, and optionally defines output tabs from the options page. If no tabs defined, output renders as a single "Result" tab with formatted text.

### Coach Registry (`/coaches/index.js`)

Imports and exports all coach configs as an array. The core iterates this array to build UI tabs, settings toggles, and route API responses. No coach is directly imported by core code.

## Phase 1: English Coach

### System Prompt Behavior

Analyzes text and returns structured JSON (no markdown). The prompt instructs:
- Use American or British English per user setting
- Target IELTS level per user setting
- Max 2 items per array (grammar, vocabulary, pronunciation). Empty array if nothing to flag.
- For pronunciation: include IPA, syllable stress, AND Color Vowel association

### Output Schema

```json
{
  "corrected": "corrected version of the message",
  "is_perfect": false,
  "grammar": [
    { "original": "phrase", "fix": "correction", "explanation": "why" }
  ],
  "vocabulary": [
    { "original": "word", "suggestion": "better word", "reason": "why" }
  ],
  "pronunciation": [
    {
      "word": "focus",
      "phonetic": "/ˈfoʊ.kəs/",
      "stress": "first syllable",
      "color_vowel": { "sound": "/oʊ/", "color": "pink", "keyword": "ROSE" },
      "tip": "Stretch the OH sound — think ROSE"
    }
  ],
  "practice_prompt": "follow-up question to continue practicing"
}
```

### Tabs

| Tab | Renders | Content |
|-----|---------|---------|
| Corrected | Single text block | The corrected version of the message |
| Grammar | List of cards | original → fix, with explanation |
| Vocabulary | List of cards | original → suggestion, with reason |
| Pronunciation | List of cards | IPA + stress + Color Vowel (color-coded) in one card per word |
| Practice | Single text block | A follow-up prompt for the user to practice |

### Color Vowel Integration

The pronunciation tab renders Color Vowel associations inline:
- Each pronunciation card shows the word, IPA, stress pattern, and the Color Vowel keyword with its associated color as a visual badge/highlight
- The 14 English vowel sounds each map to a color + keyword pair (e.g., BLUE MOON /uː/, GREEN TEA /iː/, ROSE /oʊ/)
- All generated by the AI via system prompt instructions — no external dictionary needed

### English Coach Settings

| Setting | Type | Options | Default |
|---------|------|---------|---------|
| IELTS Target Level | Select | 6.0 / 7.0 / 8.0 | 8.0 |
| English Variant | Select | American / British | American |

## Panel Behavior

### Position Modes (User Configurable)

| Mode | Behavior |
|------|----------|
| Anchored (default) | Panel appears near the focused input field |
| Fixed corner | Panel opens in bottom-right corner of viewport |
| Toolbar popover | User clicks extension icon, panel opens as popover with text from active input pre-loaded |

Stored in `chrome.storage.sync`. User selects in options page.

### Text Capture

- **Auto-capture**: when user triggers Thinkmate, text is grabbed from the currently focused input/textarea/contenteditable element
- **Editable text area**: panel always shows an editable text area with the captured text. User can modify or paste new text before sending.
- If no input is focused, the text area starts empty for manual entry.

### Trigger Button

- Subtle floating button appears near the active input field (unless in Toolbar mode)
- Debounced detection: 500ms after focus/typing before button appears
- Button shows Thinkmate icon
- Click opens the panel

### Result Actions

Each coach result section gets two buttons:

- **Copy** — copies the result text to clipboard
- **Apply** — writes the corrected/result text back into the original input field
  - For standard `<input>` and `<textarea>`: direct value replacement + dispatch input event
  - For `contenteditable`: set innerHTML/textContent + dispatch input event
  - **Fallback**: if apply fails (complex rich editor), copy to clipboard instead + show toast "Copied to clipboard instead"

### Panel UI

- Dark floating panel, clean minimal design
- "Thinkmate" branding in panel header
- Coach selector tabs at top (only enabled coaches shown)
- Word/char count displayed before sending to API
- Result sections rendered per coach config's `outputSchema`
- Smooth open/close animation
- Dismissible (click outside or close button)

### Shadow DOM

All panel UI injected inside a Shadow DOM root attached to a `<div>` appended to `document.body`. Fully isolates Thinkmate CSS from host page and vice versa. `panel.css` is loaded inside the shadow root.

## Provider & Model Configuration

All API calls go through `background.js` service worker to avoid CORS issues.

### Gemini

| Field | Value |
|-------|-------|
| Auth | API key (as URL parameter) |
| Endpoint | `generativelanguage.googleapis.com/v1beta/models/{model}:generateContent` |
| Default model | `gemini-2.0-flash` |
| Model discovery | Hardcoded list (free tier) |
| Notes | Free tier, no billing needed |
| Get key | aistudio.google.com |

### OpenRouter

| Field | Value |
|-------|-------|
| Auth | Bearer token in Authorization header |
| Endpoint | `openrouter.ai/api/v1/chat/completions` |
| Default model | `meta-llama/llama-3.3-70b-instruct:free` |
| Model discovery | Fetch from `/api/v1/models`, cache in chrome.storage for 24 hours |
| Notes | OpenAI-compatible, 100+ models, free models available with `:free` suffix |
| Get key | openrouter.ai |

### Ollama

| Field | Value |
|-------|-------|
| Auth | None |
| Endpoint | `{baseUrl}/api/chat` |
| Default base URL | `http://localhost:11434` |
| Model discovery | Fetch from `/api/tags`, populate dropdown |
| Notes | Local, fully private, no API key |

### API Client (`/core/api.js`)

Provider-agnostic interface. Content script sends `chrome.runtime.sendMessage({ type: "analyze", provider, model, systemPrompt, userText })` to background.js. Background routes to correct provider, normalizes response to `{ content: string }`.

All responses are parsed as JSON (per coach's output schema) and passed to the rendering engine.

## Options Page

### Layout

1. **Extension branding**: Thinkmate logo + tagline
2. **Provider section**:
   - Provider selector (Gemini / OpenRouter / Ollama)

   - Optional: define output tabs (key, label, type)
   - Save / Delete custom coaches

## Error Handling

| Scenario | Behavior |
|----------|----------|
| No API key configured | "Configure your AI provider" card in panel with button to open options |
| API call fails | Friendly error message in panel with retry button |
| Apply to input fails | Copy to clipboard + toast "Copied to clipboard instead" |
| Ollama not reachable | "Can't reach Ollama at {url}" with setup hint |
| Invalid JSON from AI | "Unexpected response format" with raw text shown and retry button |
| Rate limited | "Too many requests — please wait a moment" with auto-retry after delay |

## Constraints

- Manifest V3
- Vanilla JS only — no frameworks
- All API calls through background.js service worker
- API keys never logged or transmitted outside target provider
- Shadow DOM for all injected UI
- Debounce text detection (500ms)
- Graceful error handling with friendly messages
